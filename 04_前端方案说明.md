# 前端方案说明

可以参考 `panda_factor` 的前端建设思路，但不建议直接复用其现成静态页面。

## 可以借鉴的部分

- 页面结构：列表页、详情页、日志页、任务进度页
- FastAPI + 静态前端部署方式
- API 驱动的单页应用模式
- 结果可视化思路，例如趋势状态、关键位、风险标签

## 不建议直接照搬的部分

- `panda_factor` 的页面围绕因子分析与回测图设计，信息结构不适合石论
- 石论更需要“单票决策卡片 + 结构摘要 + 风险提示 + 证据链”
- 一期无需做复杂编辑器或大而全工作台

## 石论一期建议页面

### 1. 数据导入页

- 上传 CSV
- 查看导入结果
- 查看清洗错误

### 2. 快照任务页

- 启动批量快照任务
- 查看进度和日志

### 3. 单票分析页

- 输入股票代码与日期
- 展示趋势、结构类型、关键位、动作建议、风险标签、解释文本

### 4. 复盘样本页

- 查看案例快照
- 标记误判原因

## 一期前端技术建议

- 前端框架：Vue 3 或 React
- 部署方式：和后端同仓，FastAPI 提供 API，前端打包后走静态资源
- 一期不做复杂权限和多人协作

## 一期页面优先级

1. 单票分析页
2. 数据导入页
3. 任务日志页
4. 简单复盘页
