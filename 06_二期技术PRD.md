# 石论交易决策系统 - 二期技术 PRD

版本：V1.0

阶段：二期

目标周期：10-14 周

## 1. 背景与问题

一期完成了石论交易系统的基础闭环，已经能够稳定输出单票级别的结构分析、风险提示、候选建议与快照结果。

但从“能分析”走向“能证明、能执行、能迭代”，当前系统仍有几个明显缺口：

### 1.1 研究证明不足

- 现有输出更像单票分析结果，缺少严格的收益率、alpha、因子有效性验证框架
- 当前 `score`、`flag`、概率字段较多，但尚未系统证明哪些字段真正有预测力
- 缺少样本内/样本外、walk-forward、分市场阶段、分行业等验证体系

### 1.2 执行闭环不足

- 当前系统更擅长回答“这只票值不值得做”，还没有形成完整的组合策略表达
- 股票池、持仓上限、仓位控制、交易成本、涨跌停与停牌约束仍不完整
- `invalidation_level` 虽然已经生成，但尚未成为统一可回测的减仓/止损执行规则

### 1.3 聚宽落地方式需要重构

- 现有项目无法原样整体迁移到聚宽运行
- 聚宽更适合承载单文件、轻策略、偏执行型的量化脚本，而不是完整本地工程
- 因此二期必须把“研究型回测”和“聚宽执行型回测”拆成两条线

### 1.4 策略层还需要继续深化

- 当前系统对因子、量价、筹码、卖点与持有体系的拆分还不够明确
- 手工评分项较多，容易出现局部有效但整体不稳的问题
- 市场与板块变量更多是辅助信息，还没有成为真正的上层过滤器

### 1.5 工程支撑需要服务策略，而不是喧宾夺主

- 本地研究、训练、评估、快照、版本记录仍需要进一步规范
- 后续模型训练调度、批量回测、日报归档需要最小云端支撑
- 但二期不应走向“先搭大平台、再做策略”的重工程化路径

## 2. 二期目标

二期目标不是把系统做得更复杂，而是让石论成为一个：

“能研究、能证明、能执行、能迭代”的轻量量化策略系统。

围绕这个目标，二期需要完成以下五项核心任务：

### 2.1 建立本地研究型回测体系

形成一套在本地独立运行的研究平台，用于验证：

- 因子是否有效
- 信号是否稳定
- 策略是否具备超额收益
- 哪些规则在什么市场环境下生效或失效

### 2.2 建立聚宽轻策略回测体系

在聚宽能力边界内，将石论结果层抽象成单文件、可运行、可交易约束的简化策略，用于验证：

- 信号能否转成交易动作
- 最基本的仓位管理是否可执行
- 轻量抽象后是否保留可接受表现

### 2.3 深化核心交易策略

对以下方向进行体系化深化，而不是继续堆叠经验规则：

- 因子挖掘
- 量价关系深化
- 筹码分布优化
- 卖点与持仓体系
- 市场与板块过滤

### 2.4 建立组合与执行层

让系统从“单票分析引擎”升级为“轻量组合策略系统”，补齐：

- 股票池
- 排序
- 仓位控制
- 风险约束
- 调仓与交易执行规则

### 2.5 建立最小工程支撑闭环

在不做过度平台化的前提下，补齐：

- 训练脚本
- 评估脚本
- 版本记录
- 定时调度
- 回测与报告归档

## 3. 范围与非范围

### 3.1 二期范围

二期纳入以下内容：

#### A. 回测与收益证明

- 本地研究型回测框架
- 单因子、多因子、组合收益分析
- 收益率、超额收益、alpha、beta、最大回撤、Sharpe、Sortino 等评估
- 时间切分验证、样本外验证、walk-forward 验证
- 分市场阶段、分行业、分风格环境的有效性验证

#### B. 聚宽轻策略落地

- 单文件聚宽策略脚本
- 从石论结果层抽象轻量策略核
- 股票池、持仓数、仓位权重、成本与滑点模拟
- 停牌、涨跌停、成交额等交易约束
- `invalidation_level` 驱动的止损或减仓规则

#### C. 策略优化

- 因子池梳理与有效性筛选
- 量价行为因子深化
- 筹码峰、套牢盘、获利盘与压力带优化
- 卖点、持有、失败退出、防守退出规则建设
- 市场和板块过滤器建设

#### D. 组合与执行

- 候选池与股票池机制
- 排序与选股机制
- 持仓数量与单票上限控制
- 风险约束与环境降仓机制
- 调仓频率与执行策略

#### E. 工程支撑

- 本地研究目录与回测入口整理
- 训练、评估、报告输出脚本
- 模型版本与特征版本记录
- 本地与云端结合的定时调度与归档

### 3.2 二期非范围

二期不纳入以下内容：

- 完整复刻缠论原教义
- 完整高频、分钟级、逐笔级自动交易系统
- 将整个石论工程原样搬迁到聚宽
- 一开始就建设大型 Java 中台或微服务集群
- 一开始就做完整多端产品化与复杂前端平台
- 无边界扩展新闻、公告、舆情、宏观等大而全输入源

## 4. 核心模块

### 4.1 本地研究型回测模块

目标定位：研究证明主战场。

核心能力：

- 单因子分析
- 多因子横截面排序分析
- 分组收益分析
- IC / RankIC 分析
- Top-N / Decile 收益分析
- 样本内与样本外验证
- Walk-forward 验证
- 分市场环境与分行业验证
- 因子漂移与版本对比分析

建议目录：

- `research/validation/`
- `research/backtest_local/`
- `research/reports/`

### 4.2 聚宽轻策略模块

目标定位：执行验证与轻量实战验证平台。

核心能力：

- 从石论结果层抽取候选池条件
- 定义排序逻辑与最大持仓数
- 定义仓位分配规则
- 接入交易成本、滑点、停牌、涨跌停与成交额限制
- 接入 `invalidation_level` 对应的减仓或退出规则
- 输出单文件策略脚本，支持聚宽独立运行

建议产物：

- `examples/joinquant_strategy_light.py`
- `scripts/export_joinquant_strategy.py`

### 4.3 因子与策略深化模块

目标定位：减少经验堆砌，提升策略表达清晰度和可验证性。

重点方向：

- 趋势强弱因子
- 结构确认因子
- 量价协同因子
- 筹码压力与真空因子
- 风险失败因子
- 市场与板块顺风因子

同时需要输出：

- 因子定义文档
- 因子有效性结论
- 因子淘汰与保留标准

### 4.4 量价与筹码深化模块

目标定位：把差异化能力沉淀成结构化规则。

重点方向：

- 放量位置与突破质量
- 缩量回踩与承接质量
- 量价背离与趋势衰竭
- 筹码峰迁移速度
- 套牢盘消化过程
- 获利盘比例与真空区分析
- 支撑带、压力带与回踩落点关系

### 4.5 卖点与持仓模块

目标定位：补齐“持有”和“退出”。

重点方向：

- 止损规则
- 减仓规则
- 趋势结束退出
- 假突破失败退出
- 高位分布后防守退出
- `invalidation_level` 驱动的统一执行规则

### 4.6 组合与执行模块

目标定位：让单票结果变成组合结果。

核心能力：

- 股票池定义
- 候选池过滤
- 多因子排序
- 风险惩罚
- 持仓数量控制
- 单票仓位上限
- 环境驱动的整体降仓
- 调仓频率控制
- 交易失败兜底逻辑

建议目录：

- `shilun/portfolio/`
- `shilun/risk/`
- `shilun/backtest/`

### 4.7 工程与调度模块

目标定位：支撑研究、回测、训练和归档。

建议分工：

#### 本地优先

- 日常研究脚本
- 因子验证脚本
- 回测分析脚本
- 策略调参实验
- 快速人工复盘

#### 云端支撑

- 定时训练任务
- 定时批量回测
- 结果归档
- 模型与报告存储
- 对外查询服务部署

#### Java 适合承担的中长期职责

- 稳定任务编排入口
- 统一配置与版本管理
- 查询接口与服务治理
- 任务状态追踪与审计记录

说明：

二期不要求 Java 平台先行落地，但要提前把职责边界想清楚，避免后续 Python 原型难以迁移。

## 5. 里程碑

### 里程碑 M1：回测双线拆清

目标：

- 明确本地研究型回测与聚宽轻策略回测的边界
- 建立本地回测目录和统一入口
- 明确聚宽轻策略的策略核抽象方式

完成标志：

- 二期回测总方案定稿
- 本地研究回测骨架可运行
- 聚宽轻策略最小脚本可输出

### 里程碑 M2：策略信号深化

目标：

- 完成因子池梳理
- 完成量价与筹码核心规则深化
- 完成卖点与持仓体系初版

完成标志：

- 形成第一版因子池文档
- 形成第一版量价/筹码优化规则
- 形成第一版退出规则与持仓规则

### 里程碑 M3：组合与执行闭环

目标：

- 补齐股票池、排序、仓位和交易执行约束
- 让石论结果能真正组织成轻量组合策略

完成标志：

- 最大持仓数、单票上限、环境降仓机制可运行
- 调仓频率、交易成本、滑点、涨跌停与停牌约束接入
- `invalidation_level` 可在回测与执行中生效

### 里程碑 M4：研究证明与工程闭环

目标：

- 输出阶段性收益证明
- 建立训练、评估、版本、调度与归档最小闭环

完成标志：

- 输出阶段性收益分析报告
- 输出因子有效性报告
- 训练与评估脚本固化
- 定时任务与产物归档可用

## 6. 验收标准

### 6.1 回测验收

- 本地研究型回测可以稳定输出收益率、alpha、回撤、IC、分组收益等核心指标
- 能够完成样本内/样本外以及 walk-forward 验证
- 聚宽轻策略能够以单文件形式独立运行
- 聚宽脚本中已接入最基本的股票池、仓位、成本、涨跌停、停牌等执行约束

### 6.2 策略验收

- 当前核心结果字段已拆分为可解释的因子池
- 至少形成一版量价深化规则、一版筹码深化规则、一版卖点与持仓规则
- `invalidation_level` 已从分析字段升级为可执行的风险控制字段
- 市场与板块过滤已能影响候选筛选、排序或仓位控制

### 6.3 组合验收

- 系统不再只输出单票建议，而是能输出候选池、排序结果和组合持仓建议
- 最大持仓数、单票上限、环境降仓等机制可运行并可回测
- 调仓节奏和交易失败兜底逻辑具备明确规则

### 6.4 工程验收

- 本地研究、训练、评估、报告输出入口清晰
- 回测、研究、训练结果具备版本记录
- 关键任务支持定时调度
- 本地与云端职责边界明确，不再混杂

### 6.5 文档验收

- 新人首次阅读文档后，能够理解系统分层、运行原理、回测双线和二期目标
- 二期 PRD、项目结构说明、聚宽回测说明三份文档口径一致
- 回测、执行、因子、仓位、退出等关键概念都有明确文字定义

## 7. 二期一句话原则

二期不是继续堆更多规则，而是把石论从“复杂的单票分析系统”，推进到“轻量但可证明的量化策略系统”。
