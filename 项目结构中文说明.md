## 先用一句话理解这个系统

当前石论系统可以先理解成：

“输入一只股票在某一天之前的一段行情数据，系统先算趋势/波动/量价/结构/筹码等特征，再综合成统一 `snapshot`，最后给出动作建议、关键位、风险提示和解释文本。”

如果只看主链路，最重要的就是：

`bars -> features -> structure -> model -> execution -> snapshot -> decision -> explanation`

其中：

- `bars`：原始行情数据
- `features`：趋势、波动、量价、入场等基础特征
- `structure`：分型、笔、线段、中枢、背驰、事件
- `model`：延续概率、突破概率、风险等级、入场概率
- `execution`：执行分、仓位建议、支撑/压力/入场带
- `snapshot`：把前面所有关键信息装成统一结果对象
- `decision`：把 `snapshot` 变成可读的动作标签
- `explanation`：把结果渲染成解释文本

## 当前主流程是怎样跑的

如果只追主链路，可以按下面顺序理解：

1. 入口
   - API 入口：`shilun/api/__init__.py`
   - 本地入口：`ShilunPipeline.run(...)`
   - 批量入口：`SnapshotJob.run(...)`

2. 数据准备
   - 拉取或接收 `bars`
   - 标准化字段，保证至少有 `ticker/date/open/high/low/close/volume/amount`

3. 基础特征计算
   - `shilun.indicators` 计算趋势、ATR、波动和量能
   - `entry_features.py` 计算量价模式与阶段分

4. 结构识别
   - `shilun.structure` 内部完成 `pivot -> bi -> segment -> center -> divergence -> event`

5. 结构评估
   - `StructureAssessmentEngine` 把结构翻译成：
     - 结构阶段
     - 确认状态
     - 支撑位
     - 压力位
     - `invalidation_level`

6. 模型预测
   - 输出延续概率、突破概率、风险等级、入场概率

7. 执行层
   - `ExecutionEngine` 输出执行分、目标仓位、支撑带、压力带、入场带

8. 决策与解释
   - `ActionMapper` 输出动作标签
   - `PromptRenderer` 输出解释文本

9. 对外结果
   - `result.snapshot`
   - `result.decision`
   - `result.explanation`

## `invalidation_level` 是从哪里来的

这是一个很关键的字段，因为它关系到“当前乐观判断在什么价位失效”。

当前生成链路是：

1. `StructureAssessmentEngine`
   - 先根据结构状态推导 `support_level`
2. `StructureAssessmentEngine`
   - 再调用 `_derive_invalidation(...)`
   - 基于 `support_level`、ATR 和支撑缓冲比例生成 `invalidation_level`
3. `pipeline.py`
   - 把 `structure_assessment.invalidation_level` 写进 `execution_snapshot`
   - 再写入最终 `snapshot`
4. `decision` / `llm_bridge`
   - 把它用于动作摘要和解释文本

当前要特别注意：

- `invalidation_level` 已经在主流程里生成
- 但在聚宽模板策略里，还没有真正接成自动止损/减仓动作

## 当前回测到底接到什么程度

这是最容易误解的地方。

目前已经完成的是：

- 聚宽行情 `get_price(...)` -> 石论 `bars` 的字段适配
- 石论分析结果 -> 目标仓位的最小映射
- 一个可参考的聚宽模板策略
- Mongo 候选池组合回测
- 日频净值回放、alpha/beta、Sharpe、回撤和换手
- 行业、候选标签和市场状态归因

目前还没有完成的是：

- 完整股票池过滤
- 涨跌停、停牌、新股和成交额约束
- `invalidation_level` 驱动的自动止损/减仓
- 更严格的样本外收益证明和 walk-forward 验证

所以更准确的说法是：

- “最小聚宽接入骨架已实现”
- “本地组合回测闭环已进入收益证明阶段，但交易约束层还要继续补齐”
