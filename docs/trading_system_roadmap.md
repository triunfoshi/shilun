## 一、系统分层架构

交易决策由三层串联，不能跳层：

```
PART1 大盘层
  → 今天能不能做？仓位上限是多少？
  → 输出：进攻 / 持有 / 防守 / 空仓

PART2 板块层
  → 主线是哪里？哪个板块是当前资金聚焦？
  → 输出：趋势板块排名 + 龙头/中军候选池

PART3 个股层
  → 在主线板块里，今天哪只票有合适买点？
  → 输出：今日可盯票列表（带入场形态、质量分、买卖点）
```

三层相互制约：PART1 不允许进攻 → PART3 候选即使信号再好也不开仓；板块不是主线 → 个股信号降权。

**核心逻辑：`_build_pattern_forecast(index_frame, levels)`**

识别今日形态（按优先级检测）：

| 形态 | 检测条件 | 信号性质 |
|---|---|---|
| 放量突破 | close > pressure_1 + 量比≥1.5x + 收在振幅上方70% | 🟢 看多 |
| 放量破位 | close < support_1 + 量比≥1.3x + 跌幅≥0.5% | 🔴 看空 |
| 爆量上影 | 上影比>40% + 量比≥1.3x | 🟡 预警 |

| 强势大阳 | 涨幅≥1.5% + 收在振幅上方75% | 🟢 看多 |
| 缩量整理 | 量比<0.85 + 振幅<0.5% | ⚪ 中性 |
| 弱势回落 | 收跌但守支撑 | 🟠 偏弱 |
| 跌破支撑 | close < support_1（量不足构成放量破位） | 🔴 看空 |
| 小阳修复 | 涨幅 0.1%~1.5% | ⚪ 中性 |

每种形态输出：
- 今日形态名称 + 描述（2-3句）
- 明日 2-3 个关键价位（每个含：名称/具体价/支撑or压力/重要性/操作指引）
- 明日一句话核心关注

**与用户提供的参考代码对应关系：**

用户提供的 `volume_breakout` YAML 定义了：
- 阻力位识别（20日高点或前期震荡平台顶部）
- 量能确认（>2x 均量）
- 价格确认（强势收盘，在振幅上方30%）
- 次日验证逻辑（"真突破 vs 假突破"的明日关键位）

这是形态库中"放量突破"的完整规则。后续应扩充为完整形态库。

- 中期主线（20-60日）
- 长线主线（60-120日）

PART3 排序时，中期/长线主线的板块享有额外加权。

### 4.2 待实现：突破定义精化

用户原话："突破当前的定义是什么？是否有7%以上的放量大阳线？"

批次A只做到了涨幅≥3%，但用户认为真正的突破（特别是首次突破关键位）需要更大的力度：

```
真突破信号（严格版）：
- 突破日涨幅 ≥ 7%（接近涨停才算放量大阳）
- OR 涨幅 3-7% 但量比 ≥ 2.0x（量能更极端）
- 收盘在振幅上方 70%（强势收盘）
- 并非连续涨停期（炸板风险）

观察突破（宽松版）：
- 涨幅 3-7% + 量比 1.5-2.0x
- 标注为"待确认"，等次日验证
```

### 4.3 待实现：回踩定义精化

用户原话："回踩的定义是什么？从5日线上方百分之多少回踩了，并在什么价位止跌，真正的买点是否是回踩之后的上升确认？"

```
完整回踩判断流程：
1. 股价曾站上 MA5 × 1.05 以上（有过足够利润空间）
2. 随后回踩，幅度为从峰值计算下跌 5-15%（过深则破坏结构）
3. 回踩触碰 MA5 ± 2% 区间时缩量（量比 < 0.8x）
4. 次日收盘重新站上 MA5 → 这才是真正买点
5. 买点当日量能需适度放大（量比 > 1.0x，不需要爆量）
```

当前代码只做到了步骤3-4，缺少步骤1的"峰值幅度"验证和步骤2的"回踩深度"控制。

### 4.4 待实现：择时系统（最核心的缺口）

用户原话："从第三点往下细挖就是择时系统的搭建，我们的买卖点是多少，目标价位，确认点，加仓点，支撑点，破位点分别是多少？"

**择时系统应输出的字段（每只候选票）：**

```
entry_point:     实际建仓价（当日收盘）
confirm_point:   确认点（次日收盘，用于验证信号是否成立）
target_price:    目标价（近期压力位 or 前高）
add_point:       加仓点（突破确认后继续放量的进场价）
stop_loss:       止损位（跌破 MA5 × 0.98，当前已有）
breakdown_level: 破位点（不同于止损：是趋势终止的判断线，通常是 MA20 或关键支撑）
profit_space:    盈亏比（target - entry / entry - stop_loss）
```

**示例（用中国巨石 600176.SH 举例）：**
```
entry_point:     49.02（MA5 回踩确认日收盘）
confirm_point:   明日收盘能否守住 MA5
target_price:    前高 56.00（前期平台顶部）
add_point:       突破 52.00（前期震荡区间上沿）放量后
stop_loss:       48.04（MA5 × 0.98 = 49.02 × 0.98）
breakdown_level: 41.72（MA20，趋势破坏线）
profit_space:    (56 - 49) / (49 - 48) = 7x（示意，实际需计算）
```

**实现路径：**
- 目标价：取近20日内最高价附近的关键阻力（用 `rolling_high_20` + 量价结合识别）
- 加仓点：在目标价下方一定距离，或突破前高的当日收盘
- 破位点：MA20 × 0.99（给一点容忍）

### 4.5 待实现：大盘×板块增益传入 PART3

**问题：** 当前 PART3 排序只看个股的 `entry_quality`，完全没有大盘状态和板块主线的权重。导致化纤板块的票可能排在半导体前面。

## 五、Factor Lab PDF 借鉴点

> 参考：task-262-中国巨石-600176.SH_股票圆桌讨论.pdf

### 5.1 双分制度 → 启示

Factor Lab 将评分拆为两个独立维度：

| 维度 | 窗口 | 含义 |
|---|---|---|
| 近期强度 | 20日 | 趋势是否在走 |
| 最新交易日状态 | 1日（50=中性）| 今天的量价行为 |

**对我们的启示：** `entry_quality` 不应只是单一分数，应拆成：
- `trend_quality`：趋势维度（MA结构 + 20日回报 + 板块位置）
- `timing_quality`：择时维度（今日量价行为 + 是否在买点区间）

两个维度分开展示，用户能更清楚地看到"这票趋势好但今天不是好时机"。

### 5.2 三档防守线 → 止损体系

中国巨石案例中三条防守线：MA5 / MA20 / MA50

对应我们的止损体系：
- 短线止损：跌破 MA5 收盘（当前已实现）
- 中线止损：跌破 MA20（破坏主升段，减仓信号）
- 趋势止损：跌破 MA50（趋势终止）

应将这三档都输出到 PART3 候选里（目前只有 MA5 × 0.98）。

### 5.3 量价结构 → 信号的有效性过滤

Factor Lab 核心：量比 × 方向的组合判断量价是否共振。

| 量比 | 方向 | 含义 |
|---|---|---|
| 放量 | 涨 | 量价共振，有效信号 |
| 放量 | 跌 | 量价背离，卖盘主动 |
| 缩量 | 涨 | 惜售，形态健康但力度有限 |
| 缩量 | 跌 | 主动抛压不大，支撑有效 |

这个矩阵应内化到每个形态的判断条件里，不只是事后的 `risk_flags`。
