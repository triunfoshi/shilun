## 二、用户提供的战法基础代码（volume_breakout）

这是用户作为"波浪理论基础代码"给出的参考规则，格式为 YAML + 自然语言指令：

```yaml

# Price breaks above resistance on heavy volume (>2x average).

name: volume_breakout
display_name: 放量突破
description: 检测放量突破阻力位信号。适用于股价接近已知阻力位时。
category: trend
core_rules: [1, 2, 3]
required_tools:
  - get_daily_history
  - analyze_trend
  - get_realtime_quote
aliases: [放量突破, 突破]
default_priority: 30
market_regimes: [trending_up]

instructions: |
  **放量突破（Volume Breakout Strategy）**

  突破判定标准：

  1. **阻力位识别**：
     - 使用 `analyze_trend` → resistance_levels 获取阻力位。
     - 通常为 20 日高点或前期震荡平台顶部。

  2. **量能确认**（关联理念3：效率优先）：
     - 当日成交量 > 5 日均量的 2 倍。
     - 使用 `get_realtime_quote` → volume_ratio > 2.0 确认。
     - 使用 `get_daily_history` 计算均量进行交叉验证。

  3. **价格确认**（关联理念1：严进策略）：
     - 收盘价必须站上阻力位。
     - 收盘应在当日振幅上方 30%（强势收盘）。
     - 突破后乖离率检查：仍需 < 5%，避免追高。

  4. **后续验证**（如有数据）：
     - 次日开盘应在突破位之上，区分真突破与假突破。

  5. **风险过滤**（关联理念5：风险排查）：
     - 通过 `search_stock_news` 检查无重大利空。
     - PE 不应过高（避免泡沫型突破）。

  评分调整：
  - 放量突破确认：sentiment_score +12
  - 突破伴随板块共振（板块也走强）：额外 +5
  - 理想买点设在突破位附近，止损设在突破位下方 3%。
  - 在 `buy_reason` 和 `volume_analysis` 中注明"放量突破"。
```

**用户说明原话：** "可以给到你一个波浪理论的基础代码"，意指这是用来建立形态检测库的模板规范，volume_breakout 是第一个形态，后续应扩充为完整的形态库。

## 三、Factor Lab PDF 完整分析摘录

**文件：** `task-262-中国巨石-600176.SH_股票圆桌讨论.pdf`  
**主题：** 中国巨石（600176.SH）短线圆桌讨论，Factor Lab 双分制度分析框架

### 3.1 双分制度框架

Factor Lab 将股票评分拆为两个完全独立的维度：

**近期强度分（20日窗口）**

| 子因子 | 权重 | 说明 |
|---|---|---|
| 区间涨幅（20日） | 30% | 最近20日的绝对收益 |
| 5日/10日动量 | 15% | 短期动量延续性 |
| 均线结构 | 25% | MA5/MA10/MA20 的多头排列程度 |
| 相对20日高点 | 15% | 距离近期高点的回撤幅度 |
| RSI | 15% | 相对强弱指数（过热预警） |

**最新交易日状态分（单日）**

- 基准值 50 = 中性（无明显多空信号）
- 大于 50 = 当日量价结构偏多
- 小于 50 = 当日量价结构偏空
- 与"近期强度分"完全独立，避免把历史强势错认为今日买点

**中国巨石当日数据（PDF 中的具体数字）：**
- 近期强度：84/100（历史分位偏高）
- 最新交易日状态：38/100（低于中性，当日量价偏弱）
- RSI：75.14（高位）
- 量比：1.49（放量）
- 当日涨跌：下跌（放量下跌 = 量价背离）
- 前日行为：涨停（+10%），次日下跌 2.01% = 炸板

### 3.2 三档防守线

PDF 中明确给出中国巨石的三条防守线：

| 防守线 | 价格 | 对应均线 | 意义 |
|---|---|---|---|
| 短线防守 | 49.02 | MA5 | 跌破短线止损 |
| 中线防守 | 41.72 | MA20 | 跌破主升段破坏 |

### 3.3 量价结构分析框架

Factor Lab 的核心是"量比 × 方向"的组合矩阵：

| 量比 | 价格方向 | 信号含义 |
|---|---|---|
| 放量（>1.5x） | 上涨 | 量价共振，有效看多 |
| 放量（>1.5x） | 下跌 | 量价背离，卖盘主动，看空 |
| 缩量（<0.8x） | 上涨 | 多头惜售，形态健康 |
| 缩量（<0.8x） | 下跌 | 主动抛压不大，支撑有效 |

**已实现：** 批次A已加入"量价背离"风险标记（volume > volume_ma5×1.2 且 close < prev_close）。  
**未实现：** 这个矩阵应内化到每个形态的判断条件里，而不只是事后标记。

### 3.4 ADR 涨跌比阈值

PDF 中提到的市场温度参考指标 ADR（涨跌比）：
- ADR > 1.2 = 市场偏热，高温
- ADR 0.5-0.8 = 市场冷却
- ADR < 0.5 = 市场冻结，极端情绪

### 3.5 炸板识别

炸板（涨停次日大跌）是动量衰竭的极端信号：
- 中国巨石案例：前日涨停（+10%），次日下跌 2.01%
- 意义：强势动量在最极端点位出现主动抛售，动力已耗尽
- 我们的实现：批次A加入了"涨停炸板"风险标记（prev_pct ≥ 9.5% 且 today < prev_close）

## 四、交易系统完整设计思考

### 4.1 系统三层架构

用户的核心设计哲学（从最早讨论中提炼）：

```
第一层：大盘权限（PART1）
├── 今天能做什么？能开仓吗？最大仓位是多少？
└── 输出：进攻 / 持有 / 防守 / 空仓

第二层：板块动向（PART2）
├── 当前资金在哪个板块？哪里是主线？
└── 输出：趋势板块排名 + 龙头/中军候选

第三层：个股择时（PART3）
├── 在主线板块里，今天哪只票有买点？
└── 输出：今日可盯票（含买卖点结构）
```

**关键约束：三层必须串联，不可跳层。**  
PART1 不允许开仓 → PART3 信号再好也不做；板块不是主线 → 个股信号降权。

### 4.2 5日线战法完整定义

**用户原始描述（第一次提出时的原话）：**

> "另一个是结合大盘，板块，再结合自己的策略战法，选择股票并根据量价还有筹码，分析买卖点，比如我的策略是5日线战法，识别出趋势板块和龙头股票后，在股票多头并列并且突破5日线进行买入，只做三种形态（突破后确认，五日线回调突破，缩量上涨）"

**三种入场形态的精确定义（综合用户5点反馈）：**

#### 形态1：突破确认

用户问："突破当前的定义是什么？是否有7%以上的放量大阳线？"

完整定义：
- 突破日：涨幅 **≥ 7%**（接近涨停，真正的放量大阳）
- 或宽松版：涨幅 3%-7% 但量比 ≥ 2.0x（量能极端弥补涨幅不足）
- 收盘在振幅上方 70%（强势收盘，非高开低走）
- 量比 ≥ 1.5x（放量确认）
- 突破当日非买点，次日开盘守住突破位才是买点

**当前实现状态：** 批次A做到了涨幅 ≥ 3% + 量比 ≥ 1.5x，但 ≥ 7% 的严格版尚未实现。

#### 形态2：回踩MA5

用户问："回踩的定义是什么？从5日线上方百分之多少回踩了，并在什么价位止跌，真正的买点是否是回踩之后的上升确认？"

完整定义：
1. 股价曾在 MA5 上方至少 5%（`close ≥ MA5 × 1.05`，有过足够利润空间）
2. 随后回踩：从峰值的回撤幅度在 5%-15% 之间（过深则破坏结构）
3. 回踩触碰 MA5 ± 2% 区间时缩量（量比 < 0.8x，多头不急于抛售）
4. 次日收盘重新站上 MA5 → **这才是真正买点**
5. 买点当日量能适度放大（量比 > 1.0x，不需要爆量）

**当前实现状态：** 批次A实现了步骤1（`had_peak`：8日内曾高于 MA5×1.05）+ 步骤3-4，但步骤2（回踩深度5-15%范围限制）尚未实现。

#### 形态3：缩量上涨

- 最近3日连续收涨（closes[0] < closes[1] < closes[2]）
- 量能逐日收缩（多头惜售特征）
- 均线收拢（MA5 - MA20 < 3%）
- 等待放量突破时入场，本身是"待确认"状态，不是立即买点

### 4.3 择时系统：当前最大缺口

用户原话：
> "上述这些是趋势形态的确认，从第三点往下细挖就是择时系统的搭建，我们的买卖点是多少，目标价位，确认点，加仓点，支撑点，破位点分别是多少？比如附图里排名第二的巨石，今天是否是好的开仓点，不是的话如何观察等待？我们的排名是按照开仓优先级？还是本身股票成长预期？"

**择时系统需要输出的完整字段：**

```
entry_point:     实际建仓价（当日收盘，形态确认日）
confirm_point:   确认点（次日收盘，验证信号是否成立）
target_price:    目标价（近期关键压力位 or 前高）
add_point:       加仓点（趋势成立后的再次进场价）
stop_loss_1:     短线止损（MA5 × 0.98，当前已有）
stop_loss_2:     中线止损（MA20 × 0.99，主升段判断线）
stop_loss_3:     趋势止损（MA50 × 0.99，趋势终止线）
breakdown_level: 破位点（与止损不同：是趋势性判断，通常 = MA20）
profit_space:    盈亏比 = (target - entry) / (entry - stop_loss_1)
```

**以中国巨石为例（PDF 数据）：**
```
entry_point:     49.02（MA5 回踩确认日收盘）
confirm_point:   次日收盘守住 MA5（49.02 附近）
target_price:    56.xx（前期高点区间）
add_point:       突破 53.xx（前期震荡区间上沿）放量后
stop_loss_1:     48.04（MA5 × 0.98）
stop_loss_2:     41.31（MA20 × 0.99 = 41.72 × 0.99）
stop_loss_3:     36.43（MA50 × 0.99 = 36.80 × 0.99）
breakdown_level: 41.72（MA20，主升段破坏线）
```

### 4.4 大盘×板块增益：排序核心问题

用户原话（完整）：
> "除此以外，还需要考虑大盘&板块增益，比如今天排名第一的泰和新材，在我眼里就比不上巨石和兆易创新，因为从4月份开始和板块共振的就是元器件和芯片半导体，因此排名还需要额外考虑板块叠加因素，也就是我们识别出来的趋势板块加成（btw，目前趋势板块参考的周期还是太短了，在我眼里元器件和半导体应该是前二位）"

**核心问题：** 当前 PART3 排序只看 `entry_quality`（纯个股信号），完全无视板块强度和大盘状态。泰和新材（化纤板块）的入场形态可能比巨石更干净，但用户判断巨石更值得优先买，因为半导体/元器件是当前主线。

**设计方案：**
```
final_score = entry_quality × market_multiplier × sector_multiplier

market_multiplier:
  进攻 = 1.2（大盘开绿灯，可以积极）
  持有 = 1.0（正常）
  防守 = 0.7（降权）
  空仓 = 0.0（强制排除）

sector_multiplier:
  top_sectors 排名 1-3 = 1.3
  排名 4-8 = 1.1
  其余 = 0.9
```

**板块趋势周期问题：** 当前 PART2 的趋势窗口约 20-40 日，导致从 4 月开始主导的半导体/元器件板块没有体现出主线地位。需扩展到 120 日识别中期主线。

## 五、波浪理论大盘分析：已实现内容

### 5.1 实现了什么（2026-06-25）

在 `part1.py` 新增了 `_build_pattern_forecast(index_frame, levels)` 函数，识别 10 种日线形态：

| 形态 | 判断条件 | 信号类型 |
|---|---|---|
| 放量突破 | close > pressure_1 + 量比≥1.5 + 收盘在振幅上方70% | 看多 |
| 放量破位 | close < support_1 + 量比≥1.3 + 跌幅≥0.5% | 看空 |
| 爆量上影 | 上影比>40% + 量比≥1.3 | 预警 |
| 回踩守支撑 | 盘中低点触支撑附近但收盘守住 | 中性 |
| 强势大阳 | 涨幅≥1.5% + 收在振幅上方75% | 看多 |
| 缩量整理 | 量比<0.85 + 振幅<0.5% | 中性 |
| 弱势回落 | 收跌但未破支撑 | 偏弱 |
| 跌破支撑 | 收盘 < support_1（量不足放量破位） | 看空 |
| 小阳修复 | 涨幅 0.1%-1.5% | 中性 |
| 震荡整理 | 无明显信号 | 中性 |

每种形态输出：
- 今日形态描述（2-3句，含量价数据）
- 明日 2-3 个关键价位（含方向/重要性/操作指引）
- 明日一句话核心关注

参数来源于 volume_breakout 战法的逻辑结构（阻力位识别 → 量能确认 → 价格确认 → 次日验证）。
