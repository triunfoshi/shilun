# 石论二期纲领文件

本文档用于统一二期方向，替代此前分散在 `开发任务清单.md`、`docs/joinquant_backtest.md`、临时讨论和零散备注中的后续规划。

二期不再继续围绕“一期是否已经跑通”展开，而是明确进入：

- 策略深化
- 回测证明
- 组合与执行
- 最小工程化支撑

二期的核心判断只有一句话：

石论不能停留在“单票分析和解释系统”，必须升级为“可验证、可回测、可执行的轻量量化策略系统”。

---

## 一、二期总目标

二期的总目标不是把系统做得更复杂，而是把以下三件事真正做实：

1. 证明石论信号是否具备稳定 alpha
2. 把石论结果层抽象为可执行的轻量策略
3. 建立支持持续训练、持续评估、持续迭代的最小工程底座

这里的重点是：

- 策略优先
- 回测优先
- 执行优先
- 平台化和云化只做必要支撑

---

## 二、二期范围边界

### 2.1 二期必须解决的问题

二期必须回答以下问题：

1. 石论里的哪些字段是真正有效的因子
2. 量价关系和筹码分布能否形成稳定交易优势
3. 当前结果层如何转成“能下单、能控仓、能回测”的轻量策略
4. 在聚宽能力边界下，如何落地一个单文件可运行策略
5. 在本地环境下，如何形成更完整的收益率、alpha、因子有效性分析体系
6. 如何把训练、评估、版本、定时任务接成最小闭环

### 2.2 二期暂不追求的事情

二期暂不以以下内容为目标：

1. 完整复刻整套缠论原教义
2. 一开始就做大型 Java 平台
3. 一开始就做完整多端产品化
4. 一开始就做超大规模云原生微服务体系
5. 把整个石论项目无改造整体迁移到聚宽

---

## 三、二期的两类回测体系

二期回测必须拆成两个部分，不再混成一个目标。

## 3.1 本地研究型回测体系

这是石论二期最核心的“收益证明”和“因子有效性分析”平台，目标不是下单，而是研究。

### 目标

回答以下问题：

1. `execution_score`、`entry_probability`、`p_continue_10d` 等字段是否有预测力
2. 量价因子、筹码因子、结构因子分别贡献了什么
3. 不同市场阶段下，哪些信号有效、哪些失效
4. 组合后是否存在稳定超额收益
5. 样本外和 walk-forward 下是否仍成立

### 本地研究型回测必须实现的分析能力

1. 单因子和多因子分组收益分析
2. IC / RankIC / 分层收益 / top-decile return
3. 收益率、超额收益、alpha、beta、最大回撤、Sharpe、Sortino
4. 时间切分验证
5. Walk-forward 验证
6. 分市场环境验证
7. 分行业验证
8. 因子稳定性分析
9. 特征漂移分析
10. 策略版本对比分析

### 本地研究型回测要提前部署的代码方向

建议新增或完善以下模块：

1. `research/validation/`
   - 因子有效性分析
   - IC 和分组收益分析
   - 横截面排序分析

2. `research/backtest_local/`
   - 本地组合回测
   - 交易成本模拟
   - 股票池过滤
   - 调仓频率测试

3. `research/reports/`
   - 输出研究报告
   - 输出图表和评估摘要

4. `shilun/portfolio/`
   - 组合排序
   - 仓位分配
   - 调仓逻辑
   - 风险约束

### 本地研究型回测的定位

它是“证明石论有没有研究价值”的主战场。

后续的模型训练、因子筛选、策略迭代，都应该以这条线为主，而不是以聚宽模板回测作为最终裁判。

## 3.2 聚宽轻策略回测体系

这一部分的目标不是迁移整个石论，而是：

在聚宽当前能力范围内，把石论结果层抽象成一个“单文件、可运行、偏执行型”的轻量策略。

### 目标

做出一个真正能在聚宽里独立运行的简化策略脚本，用于：

1. 验证石论核心信号是否能转成交易动作
2. 验证最基本的仓位控制和交易执行
3. 验证轻量化后是否仍有可接受表现

### 这一部分必须坚持的原则

1. 轻策略
   - 聚宽脚本只保留必要策略核，不尝试搬运整个石论系统

2. 重量化
   - 重点放在选股、仓位、调仓、交易约束、成本模拟、止损减仓这些量化执行问题上

3. 单文件部署
   - 最终必须能落成一个独立 Python 策略文件

### 聚宽轻策略的核心实现方向

石论要先把结果层抽象成以下几个可部署信号：

1. 入选条件
   - 哪些股票进入候选池

2. 排序条件
   - 如何从候选池中选出优先交易标的

3. 仓位条件
   - 每只股票给多少权重

4. 调仓条件
   - 每日调仓、每周调仓还是事件调仓

5. 卖出条件
   - 主动卖出
   - 失败退出
   - 减仓规则

6. 风控条件
   - `invalidation_level` / `invalidation_price`
   - 风险高时压仓位
   - 涨跌停、停牌、成交额限制

### 聚宽轻策略必须补齐的执行项

1. 股票池定义
2. 仓位控制
3. 交易成本
4. 停牌过滤
5. 涨停跌停过滤
6. 成交额门槛
7. `invalidation_level` 接成止损/减仓规则
8. 调仓节奏
9. 最大持仓数

### 聚宽轻策略建议产物

二期建议明确产出一个单独文件，例如：

- `examples/joinquant_strategy_light.py`

或者进一步提供一个导出脚本：

- `scripts/export_joinquant_strategy.py`

后者的目标是：

- 从本地石论配置和信号逻辑中抽取轻量策略核
- 输出一份聚宽可运行的单文件脚本

### 聚宽轻策略的定位

它是“量化执行验证”和“轻量实战验证”的平台。

它不负责完整研究，不负责完整训练，不负责完整因子分析。

## 四、二期策略深化主线

除了回测体系，二期真正的核心仍然是策略本身。

## 4.1 因子挖掘

二期要把石论现有结果层拆成明确因子池，而不是继续堆 `score` 和 `flag`。

重点因子方向：

1. 趋势强弱因子
2. 结构确认因子
3. 量价协同因子
4. 筹码压力/真空因子
5. 风险失败因子
6. 市场和板块顺风因子

二期要做的不是继续发明更多指标，而是回答：

- 哪些因子有效
- 哪些因子冗余
- 哪些因子在什么市场环境下有效

## 4.2 量价关系深化

二期要把量价从“指标层”升级成“行为层”。

重点研究方向：

1. 放量发生的位置
2. 缩量回踩的质量
3. 突破量能与承接质量的联动
4. 量价背离与趋势衰竭
5. 量价与收盘质量、波动压缩、阶段位置的联合判断

## 4.3 筹码分布优化

筹码是石论最值得做出差异化的部分之一，二期必须深化。

重点方向：

1. 筹码峰迁移速度
2. 上方套牢盘消化过程
3. 获利盘比例和真空区分析
4. 回踩落点与筹码密集区关系
5. 支撑带和压力带的更稳健构造

## 4.4 卖点与持仓体系

当前石论更像“买点分析系统”，二期必须补齐“持有和退出”。

重点方向：

1. 止损
2. 减仓
3. 趋势结束退出
4. 假突破失败退出
5. 高位分布后防守退出
6. `invalidation_level` 驱动的执行规则

## 4.5 市场与板块状态过滤

二期要把市场和板块从辅助变量升级成上层过滤器。

重点方向：

1. 市场阶段分类
2. 板块扩散与收敛识别
3. 弱市下降低进攻信号权重
4. 顺风环境放大核心信号

---

## 五、二期组合与执行层建设

二期的目标不是只会给单票建议，而是把信号真正组织成组合。

## 5.1 股票池建设

至少要实现：

1. 基础股票池
2. ST 过滤
3. 停牌过滤
4. 涨跌停过滤
5. 成交额门槛
6. 可选指数成分股池

## 5.2 排序与选股

至少要实现：

1. 候选池排序
2. 多因子加权排序
3. 风险惩罚排序
4. 不同市场环境下排序权重切换

## 5.3 仓位控制

至少要实现：

1. 最大持仓数
2. 单票上限
3. 试仓/标准仓/防守仓
4. 风险高时整体降仓
5. 板块过热时限制暴露

## 5.4 交易执行约束

至少要实现：

1. 调仓频率
2. 交易成本
3. 滑点
4. 涨跌停限制
5. 停牌处理
6. 无法成交时的兜底逻辑
