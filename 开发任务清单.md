# 开发任务清单

本文档用于把“缠论结构 + 统计特征 + 概率建模 + 决策解释”的完整方案拆成可执行开发任务。

这里不按 MVP 思路收缩范围，而按完整产品和完整研究系统设计。目标不是先证明能跑，而是建设一套结构清晰、定义严谨、可训练、可验证、可解释、可扩展的趋势与结构分析引擎。

---

## 一、总目标

将当前项目从“基于少量技术指标阈值的规则判断”升级为完整的多层分析系统：

1. 数据层  
   统一管理原始行情、清洗、复权、聚合、多周期数据。

2. 基础指标层  
   计算趋势、波动、量能、相对强弱、统计分布、标准化特征。

3. 缠论结构层  
   严格识别分型、笔、线段、中枢、离开、回试、背驰、级别联立结构。

4. 特征工程层  
   将基础指标与缠论结构对象组合成模型可用的扁平特征。

5. 标签与样本层  
   定义趋势延续、突破成功、结构失效、回撤风险等监督标签。

6. 模型层  
   建立状态识别模型、事件概率模型、风险模型、校准层。

7. 决策层  
   基于结构状态、概率输出和风险约束映射为交易动作建议。

8. 解释层  
   输出结构化解释、证据链、规则来源和概率来源。

9. 评估层  
   建立样本外评估、滚动验证、分市场环境验证、稳定性分析。

10. 工程层  
    实现配置化、测试化、可复现实验、可维护目录结构和文档。

---

## 二、完整方案设计

### 2.1 设计原则

完整系统必须满足以下原则：

1. 定义严格  
   所有缠论概念必须能落为数据对象和计算规则，不能停留在文字描述。

2. 统计可验证  
   所有趋势定义和结构信号都必须能通过历史样本验证，而不是只凭经验阈值。

3. 不前视  
   任何结构识别、事件标注、模型特征都不能使用未来数据。

4. 多周期联立  
   周线、日线、低周期之间必须能表达支持、冲突和共振。

5. 可解释  
   最终输出不仅要有标签，还要有结构原因、概率原因、风险原因。

6. 可替换  
   规则回退、统计模型、模型版本、特征版本都必须具备可替换接口。

7. 工程完备  
   文档、测试、配置、日志、训练流程、评估流程、模型产物都要归档。

### 2.2 分层架构设计

建议总流程如下：

```text
原始 bars
-> 清洗与统一格式
-> 多周期聚合
-> 基础指标计算
-> 分型识别
-> 笔构建
-> 线段构建
-> 中枢识别
-> 背驰识别
-> 结构事件识别
-> 结构特征提取
-> 多周期联立特征提取
-> 标签生成
-> 状态模型预测
-> 事件模型预测
-> 风险模型预测
-> 校准
-> snapshot 组装
-> 决策映射
-> 文本解释
```

### 2.3 完整输出目标

最终系统应输出以下结果：

1. 结构状态
   - 当前处于何种缠论结构阶段
   - 当前结构是否健康
   - 当前中枢是否上移/下移
   - 是否存在背驰、失败离开、结构破坏

2. 统计状态
   - 趋势强度
   - 波动环境
   - 量能确认度
   - 趋势效率
   - 相对位置强弱

3. 概率状态
   - 趋势延续概率
   - 突破成功概率
   - 短期失效概率
   - 预期收益
   - 预期回撤

4. 决策状态
   - 可跟随
   - 等待回踩
   - 只观察
   - 减仓防守
   - 避免追高

5. 解释证据
   - 结构证据
   - 指标证据
   - 概率证据
   - 风险证据

1. 真正基于多周期数据和缠论结构输出结构状态

3. 输出完整概率和风险字段

5. 样本外验证、滚动验证、校准验证全部完成
6. 测试覆盖关键结构与状态机分支
7. 文档完整，字段、流程、训练、评估均可查
8. 旧逻辑被完整兼容或正式废弃，不留灰区
